  High-Value Clinical Trials Mappings for Biobtree

  Based on the clinical trials data schema and your existing biobtree datasets, here are the most valuable
  mappings:

  1. Intervention Names → ChEMBL/DrugBank (HIGHEST VALUE!)

  Why: Links clinical trials to chemical compounds and drugs
  - From: interventions[].name (drug/compound names)
  - To: ChEMBL IDs, compound structures
  - Query Example: "CHEMBL203 >> clinical_trial" → Find all trials testing aspirin
  - Reverse: "NCT12345 >> intervention >> chembl >> uniprot" → Trial → Drug → Target

  Extraction Strategy:
  - Parse drug names from interventions
  - Fuzzy match against ChEMBL/DrugBank compound names
  - Create mapping table: {intervention_name, nct_id, chembl_id, confidence}

  2. Disease/Condition Terms → Disease Ontologies

  Why: Links trials to standardized disease classifications
  - From: Conditions mentioned in trials (not in current schema but in AACT conditions table)
  - To: Disease ontologies (EFO, OMIM, MeSH when you add DisGeNET)
  - Query Example: "disease:diabetes >> clinical_trial" → All diabetes trials
  - Reverse: "NCT12345 >> condition >> disgenet >> hgnc" → Trial → Disease → Genes

  3. Gene/Protein Mentions in Trial Text

  Why: Links trials to molecular targets
  - From: Text mining brief_summary, detailed_description, eligibility.criteria
  - To: HGNC gene symbols, UniProt IDs
  - Query Example: "HGNC:EGFR >> clinical_trial" → Trials mentioning EGFR
  - Reverse: "NCT12345 >> gene >> uniprot >> chembl" → Trial → Gene → Drugs

  Extraction Strategy:
  - NER (Named Entity Recognition) on trial text
  - Match against HGNC symbols, gene names
  - Store: {nct_id, gene_symbol, mention_context, confidence}

  4. NCT ID as Primary Identifier ✅

  Why: Universal clinical trial identifier
  - Primary Key: nct_id (e.g., NCT01234567)
  - Enables: Direct lookup and all other mappings

  5. Sponsor/Organization

  Why: Links trials to pharmaceutical companies (complements patent data!)
  - From: Sponsor information (in AACT sponsors table)
  - To: Patent assignees (from your SureChEMBL integration)
  - Query Example: "assignee:AstraZeneca >> patent >> surechembl >> chembl >> clinical_trial"
  → Company → Patents → Compounds → Trials

  6. Phase Information

  Why: Track drug development stage
  - From: phase field
  - To: Searchable attribute
  - Query Example: "phase:Phase_3 >> clinical_trial >> intervention >> chembl"
