# AlphaFold Test Dataset

## Overview

This directory contains test data for the AlphaFold Protein Structure Database integration.

AlphaFold is a database of protein structure predictions generated by the AlphaFold system developed by DeepMind. It provides highly accurate 3D structure predictions for proteins across the proteome, with per-residue confidence estimates (pLDDT scores).

## Test Data

### Source

The test data uses the **Swiss-Prot subset** of AlphaFold predictions (version 6). This is a curated subset containing only reviewed UniProt entries, providing high-quality, well-annotated protein structures.

**Why Swiss-Prot only?**
- **Size**: Swiss-Prot tar archive is 28.5 GB vs 214M entries for full AlphaFold
- **Quality**: Only reviewed, manually-annotated UniProt entries
- **Practicality**: Manageable dataset size for testing and distribution
- **Coverage**: Still includes ~550,000 high-confidence protein structures

### Dataset Statistics

- **Test Proteins**: 20 UniProt IDs (Swiss-Prot reviewed entries)
- **Source**: AlphaFold Swiss-Prot v6 (swissprot_pdb_v6.tar)
- **File Format**: Tar archive of gzipped PDB files (.pdb.gz)
- **Size**: 28.5 GB compressed, ~550K structures
- **Confidence Metric**: pLDDT (predicted Local Distance Difference Test), range 0-100
- **Reference Data Source**: AlphaFold DB API (https://alphafold.ebi.ac.uk/api)

### Extracting Reference Data

Reference data is fetched from the official AlphaFold DB REST API:

```bash
cd tests/alphafold
python3 extract_reference_data.py      # Extract all 20 IDs
python3 extract_reference_data.py 10   # Extract first 10 IDs only
```

The script:
1. Reads UniProt IDs from `alphafold_ids.txt`
2. Fetches structure predictions via `/api/prediction/{uniprot_id}`
3. Saves complete API responses to `reference_data.json`

Rate limiting: 0.5 seconds per ID (~10 seconds for all 20 IDs)

## AlphaFold Integration Architecture

### Storage Model

AlphaFold data is stored as **attributes on UniProt entries** with bidirectional cross-references:

```
UniProt Entry (e.g., A0A009IHW8)
  ├── AlphaFold Attributes:
  │     ├── global_metric: 85.17
  │     ├── fraction_pldddt_very_high: 0.525
  │     ├── fraction_pldddt_confident: 0.350
  │     ├── fraction_pldddt_low: 0.113
  │     ├── fraction_pldddt_very_low: 0.012
  │     ├── model_entity_id: "AF-A0A009IHW8-F1"
  │     └── gene: "J512_3302"
  └── Cross-references:
        └── UniProt (dataset 1) ↔ AlphaFold (dataset 30)

AlphaFold Model ID (e.g., AF-A0A009IHW8-F1)
  └── Keyword link → UniProt entry (A0A009IHW8)
```

**Key Features:**
- UniProt IDs are primary keys (e.g., `A0A009IHW8`)
- AlphaFold model IDs are keywords (e.g., `AF-A0A009IHW8-F1`)
- Both UniProt IDs and model IDs can be used to query structure data
- Bidirectional cross-references between UniProt and AlphaFold datasets

### Confidence Scoring (pLDDT)

AlphaFold provides per-residue confidence estimates using pLDDT (predicted Local Distance Difference Test):

| pLDDT Range | Confidence Level | Interpretation |
|-------------|------------------|----------------|
| **> 90** | Very High | High accuracy, well-ordered regions |
| **70-90** | Confident | Generally good backbone prediction |
| **50-70** | Low | Low confidence, potential disorder |
| **< 50** | Very Low | Should not be interpreted, likely disordered |

The `global_metric` is the average pLDDT across all residues, providing an overall structure quality score.

### Data Processing

The AlphaFold processor:

1. **Streams tar.gz archive** from EBI FTP (no full extraction needed)
2. **Decompresses nested gzip** - each `.pdb.gz` file within the tar
3. **Parses PDB files** - extracts pLDDT scores from B-factor column (positions 60-66)
4. **Calculates metrics**:
   - Global metric (average pLDDT across all atoms)
   - Confidence fractions for four confidence categories
5. **Creates entries**:
   - Stores attributes on UniProt ID
   - Creates keyword for AlphaFold model ID

### File Format Details

**Archive Structure:**
- Archive: `swissprot_pdb_v6.tar` (tar, not gzipped)
- Individual files: `.pdb.gz` (gzipped PDB files)
- Naming: `AF-{UniProtID}-F{fragment}-model_v{version}.pdb.gz`
- Example: `AF-A0A009IHW8-F1-model_v6.pdb.gz`

**PDB B-factor Column:**
- AlphaFold stores pLDDT scores in the B-factor field
- Positions 60-66 in ATOM records
- Range: 0-100 (floating point)

## Usage

### Building with Test Data

```bash
# Build with AlphaFold test data (20 structures)
./biobtree -d alphafold test

# Build with custom entry limit
# Edit conf/source.dataset.json:
#   "alphafold": { "test_entries_count": "50" }
./biobtree -d alphafold test
```

### Query Examples

```bash
# Query by UniProt ID directly
curl "http://localhost:9292/ws/entry/?i=A0A009IHW8&s=alphafold"
# Returns AlphaFold attributes with confidence scores

# Query by AlphaFold model ID (keyword lookup)
curl "http://localhost:9292/ws/entry/?i=AF-A0A009IHW8-F1&s=alphafold"
# Resolves to UniProt entry with AlphaFold data

# Get all available structure data
curl "http://localhost:9292/ws/entry/?i=A0A009IHW8"
# Returns UniProt entry with AlphaFold attributes in xrefs

# Example response:
{
  "results": [{
    "id": "A0A009IHW8",
    "dataset": 30,
    "Attributes": {
      "Alphafold": {
        "global_metric": 85.17,
        "fraction_pldddt_very_high": 0.525,
        "fraction_pldddt_confident": 0.350,
        "fraction_pldddt_low": 0.113,
        "fraction_pldddt_very_low": 0.012,
        "model_entity_id": "AF-A0A009IHW8-F1",
        "gene": "J512_3302"
      }
    }
  }]
}
```

## Test Cases

### 1. Basic Structure Lookup

**Test**: AlphaFold structures can be queried by UniProt ID
- Query UniProt ID (e.g., `A0A009IHW8`)
- Verify AlphaFold attributes exist
- Check `global_metric`, `model_entity_id` fields present

### 2. Confidence Metrics Validation

**Test**: pLDDT fractions sum to 1.0 and are in valid ranges
- Sum of four fraction fields ≈ 1.0 (within 0.01 tolerance)
- Each fraction in range [0.0, 1.0]
- Global metric in range [0, 100]

### 3. Model ID Format

**Test**: Model IDs follow expected naming convention
- Pattern: `AF-{UniProtID}-F{N}`
- Example: `AF-A0A009IHW8-F1`
- UniProt ID portion matches entry

### 4. Keyword Lookup

**Test**: AlphaFold model IDs resolve to entries
- Query by model ID (e.g., `AF-A0A009IHW8-F1`)
- Should resolve to AlphaFold dataset (ID 30)
- Returns corresponding UniProt entry with structure data

### 5. UniProt Cross-References

**Test**: Bidirectional references between UniProt and AlphaFold
- AlphaFold entries have xrefs to UniProt (dataset 1)
- UniProt queries return AlphaFold data
- Model ID queries reference back to UniProt ID

### 6. High-Confidence Structures

**Test**: Most Swiss-Prot entries have reasonable confidence
- Expect >50% with global_metric > 70
- Swiss-Prot entries are typically well-structured proteins
- Low-confidence structures are flagged appropriately

### 7. Gene Annotations

**Test**: Some entries have gene symbols
- Check `gene` field present when available
- Not all entries have gene annotations (optional field)
- Gene symbols match UniProt annotations

## Swiss-Prot vs Full Dataset

### Why Swiss-Prot Subset?

| Aspect | Swiss-Prot Subset | Full AlphaFold |
|--------|------------------|----------------|
| **Size** | 28.5 GB | ~23 TB |
| **Entries** | ~550,000 | ~214 million |
| **Quality** | Reviewed only | All predictions |
| **Annotations** | Rich metadata | Variable quality |
| **Download time** | Minutes | Days |
| **Processing** | ~2-5 minutes | Hours |

### Full Dataset Support

While the test uses Swiss-Prot, the integration supports the full AlphaFold database:

```bash
# To use full dataset, update conf/source.dataset.json:
"path": "https://ftp.ebi.ac.uk/pub/databases/alphafold/latest/UP000005640_9606_HUMAN_v6.tar"

# Or for all organisms:
"path": "https://ftp.ebi.ac.uk/pub/databases/alphafold/latest/all_pdb_v6.tar"
```

**Note**: Full dataset processing requires:
- Sufficient disk space (~50+ GB for database)
- Several hours processing time
- Streaming architecture handles arbitrary dataset size

## Performance Notes

- **Swiss-Prot test (20 structures)**: ~2.5 seconds
- **Swiss-Prot full (~550K structures)**: ~5-10 minutes
- **Memory usage**: Streaming, no full extraction needed
- **Disk usage**: Test database ~5 MB, full Swiss-Prot ~2-3 GB

## Known Limitations

1. **Version tracking**: Currently stores latest version only (v6). Historical versions not tracked.

2. **Fragment structures**: Some proteins have multiple fragments (F1, F2, etc.). Each fragment is a separate entry.

3. **Isoforms**: AlphaFold typically models canonical sequence only. Alternative isoforms may not have predictions.

4. **Confidence interpretation**: pLDDT measures local accuracy, not global fold accuracy. High pLDDT regions can still have incorrect relative orientations.

5. **PAE data**: Predicted Aligned Error (PAE) matrices are not currently stored, only summary metrics.

## Maintenance

To update the test dataset with newer AlphaFold versions:

1. Update `path` in `conf/source.dataset.json` to latest version (e.g., v7)
2. Regenerate test database: `./biobtree -d alphafold test`
3. Update test IDs: `cp test_out/reference/alphafold_ids.txt tests/alphafold/`
4. Re-run `extract_reference_data.py` to fetch new API data
5. Verify tests pass with new version
